# 固定要件一覧 (Fixed Requirements)

> このファイルはプロジェクト全体を通じて変わらない「固定方針」を記録する。  
> 実装・レビュー時に必ず参照すること。最終更新: 2026-04-11

---

## 実装方針

| # | 方針 | 詳細 |
|---|------|------|
| 1 | 採用済み機能案は固定要件 | これまで採用・実装した機能案はすべて固定要件として扱う |
| 2 | 無料機能を先に強く完成させる | 有料 / API 機能より先に無料体験の質を高める |
| 3 | API 土台を先に作る | API 必須機能は「いつでも ON にできる土台」まで先に実装し、スイッチで有効化 |
| 4 | 既存コードを全削除しない | リライトではなく最小差分での段階的強化 |
| 5 | 機能を細分化する | 後から修正しやすい単位で機能を分割・管理 |
| 6 | 基盤を壊さない | 認証・管理画面・広場・ゲスト制限・テンプレ基盤を保護 |
| 7 | スマホ最適化を最優先 | 全画面でモバイル体験を最優先に設計・実装する |
| 8 | 日本語固定を徹底 | UI・プロンプト・エラー・ドキュメント、すべて日本語 |
| 9 | 権限を分離する | admin（管理者）と headquarters（管理本部）の権限を明確に分ける |
| 10 | API 利用可視化 | 管理者のみが API 利用量・概算コストを確認できる |
| 11 | API ON/OFF 制御 | 機能単位で ON/OFF・一括指定・予算上限・自動停止・無料代替フォールバック |
| 12 | フェーズ完了でコミット | 1 フェーズ完了ごとにコミット & push する |

## 世界観 / UI 方針

| 要素 | 方針 |
|------|------|
| 全体テーマ | 明るい冒険ギルド・集会所・工房・依頼板 |
| ゲーム感 | ロゴ・見出し・称号・バッジ・主要ボタンはゲーム風デザイン |
| 本文 | 可読性重視（ゲーム風すぎない） |
| カラー基調 | 琥珀色・パーチメント・アンバーゴールド |
| フォント | Noto Sans JP (本文) + Inter (英数字) |

## ロール定義

| ロール | 説明 | 実装状態 |
|--------|------|---------|
| `guest` | 未ログイン訪問者 (1日3回制限) | ✅ JS で管理 |
| `member_free` | 無料会員 | ❌ DB は `user` のみ |
| `member_paid` | 有料会員 | ❌ 未実装 |
| `member_master` | マスター会員 | ❌ 未実装 |
| `admin` | 管理者 (サーバー操作・ユーザー管理) | ✅ |
| `headquarters` | 管理本部 (戦略・コンテンツ・分析) | ❌ 未実装 |

## フェーズ出力フォーマット

各フェーズ完了時に以下を出力すること:

1. フェーズ名
2. 現状課題
3. 実装方針
4. 変更ファイル
5. 実装内容
6. 確認手順
7. 実装結果
8. コミットハッシュ
9. push 先ブランチ
10. 次フェーズ前の注意点

## API 制御方針

```
各 API 機能には以下を必ず実装:
1. feature_flags による ON/OFF スイッチ
2. 予算上限 (月次トークン上限 or 円換算)
3. 超過時の自動停止 + 管理者通知
4. 無料代替モード (API なしで動作する基本機能)
5. 管理者ダッシュボードで利用量・コスト確認
```

## 守るべき既存基盤

- `app/auth/` — JWT 認証・ステータス管理
- `app/admin/` — 管理機能
- `app/community/` — 広場・reactions
- `app/gamification/` — XP・レベル・バッジ
- `static/js/core/guest.js` — ゲスト制限
- `app/core/feature_flags.py` — 機能フラグ
