> 最終更新: 2026-04-18

## 2. 日常運営 (admin)

### ユーザー管理
1. `/admin` → 「ユーザー管理」タブ
2. 承認待ちユーザーを確認 → 承認 / 却下
3. 問題ユーザーは「停止」操作

### トレンドデータ更新
1. `/admin` → 「🔥 トレンド管理」タブ
2. 各工房の鮮度を確認 (fresh / expiring / stale)
3. 期限切れ工房の「▶ 更新」ボタンをクリック
4. または「全工房を更新」で一括実行
- 自動更新: 毎日 03:00 JST (systemd timer)

### サービス状態確認
- `/admin` → 「⚙ サービス状態」タブで全サービスの動作モードを確認
- API接続なし = FREE (テンプレートベース) で動作中

### メンテナンスモード
1. `/admin` → 「システム設定」タブ
2. 「メンテナンスモード」トグルをON
3. 全ユーザーにメンテナンス画面が表示される

## 3. 管理本部専用操作 (headquarters)

### アクセス
- URL: `/hq` (headquarters ロールのみアクセス可)
- admin ロールは `/hq` にアクセス不可 (→ `/admin` へリダイレクト)

### KPI 概要パネル
- ユーザー数・活動量・稼働サービス数・月次コストを確認
- 「オープンインシデント」件数に注意 (0件が正常)
- Phase 18 時点: ユーザー数はDB接続後にリアルデータ表示

### Trend Room
- 全7工房のトレンドデータ鮮度を横断確認
- 「推奨アクション」欄の指示に従って更新
- クロス分析: 全工房で共通して hot なカテゴリを確認

### Incident Room

**インシデントの作成:**
1. 「+ 新規作成」ボタン
2. タイトル・深刻度 (info/warning/critical) ・説明を入力
3. 「作成する」をクリック

**深刻度の基準:**
| 深刻度 | 基準 |
|--------|------|
| info | 軽微な異常・要観察 |
| warning | サービス一部影響・要対応 |
| critical | サービス停止・即時対応必要 |

**ステータス遷移:**
```
open → monitoring → resolved
```
- `open`: 発生・未対応
- `monitoring`: 対応中・観察中
- `resolved`: 解決済み (解決内容を記録)

### Feature Flag Console

**ランタイムオーバーライド:**
- トグルスイッチで即時 ON/OFF
- `OVERRIDE` バッジ = 環境変数と異なる設定が適用中
- 「解除」ボタン = 環境変数の値に戻す
- **注意: サーバー再起動でリセットされる**

**永続的な変更:**
```bash
# .env または環境変数を更新してサーバー再起動
ENABLE_CAMPAIGN_FORGE=false  # 例: キャンペーン工房を無効化
```

### Admin 管理 (HQ 専用)
- admin / headquarters ロールのユーザー一覧を閲覧
- ロール変更: セレクトボックスで変更 → 確認ダイアログ
- **admin ロールは他の admin を変更できない** (HQ 専用権限)

## 4. 緊急時対応手順

### API コスト急増時
1. `/hq` → 「⚙ Flag Console」
2. 高コストサービスを即時無効化:
   - `WORKSHOP_MASTER_AI` → OFF
   - `ARTICLE_DRAFT_AI` → OFF
   - `IMAGE_API_CALL` → OFF
3. インシデントを作成 (severity: critical)
4. `/admin` → 「🎛 API制御」→「省エネプリセット」を適用

### 不正ユーザー対応
1. `/admin` → 「ユーザー管理」→ 対象ユーザーを「停止」
2. インシデントに記録

### メンテナンス緊急実施
1. `/admin` → 「システム設定」→ メンテナンスモード ON
2. 作業完了後 OFF
3. インシデントに作業内容を記録

## 5. 集客工房の利用 (admin/HQ)

1. `/admin` → 「🚀 集客工房」タブ
2. ツールカードをクリック → モーダルが開く
3. フォームに必要情報を入力
4. 「▶ プロンプト生成」→ 生成されたプロンプトをコピー
5. ChatGPT / Gemini / Claude に貼り付けて実行

**主要ツール:**
| ツール | 用途 |
|--------|------|
| ローンチパック | 商品リリース時の全素材を一括生成 |
| 販促カレンダー | 30日間の投稿スケジュール生成 |
| フック文ライブラリ | 記事冒頭の引き込み文を10パターン生成 |
| 販促スコアリング | 現在の販促プランを採点・改善提案 |
